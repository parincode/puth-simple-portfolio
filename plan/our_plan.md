# Plan: Data Analyst Portfolio Website

This plan outlines the creation of a modern, dark-themed portfolio website for "Parin Puth", a Data Analyst, to be hosted on GitHub Pages.

## Objective
Build a single-page, responsive portfolio website featuring a dark aesthetic, JetBrains Mono typography, a hero section, a showcase of three data projects, and contact information.

## Key Files & Context
- `index.html`: Main structure of the portfolio.
- `style.css`: Styling with a focus on dark mode, JetBrains Mono, and modern layouts.
- `script.js`: Minor interactions (e.g., smooth scrolling, project hover effects).
- `assets/`: Folder for project images/thumbnails.

## Proposed Strategy

### 1. Design & Typography
- **Theme:** Deep dark background (`#0d1117` or similar) with high-contrast text (`#e6edf3`) and subtle accent colors (e.g., Data Analyst blue or green).
- **Font:** Integrate "JetBrains Mono" via Google Fonts for a developer/data-centric feel.
- **Layout:** Flexbox/Grid for a clean, modular look.

### 2. Sections
- **Hero Section:** Big, bold "Parin Puth" with a subtitle like "Data Analyst | Insights Enthusiast".
- **Projects Section:** Three cards, each containing:
    - Project Title
    - Brief description (e.g., "Exploratory Data Analysis of X")
    - Key tools used (Python, SQL, Tableau, etc.)
    - Placeholder links to GitHub or live reports.
- **Contact Section:** Simple layout with Email, LinkedIn, and GitHub links.

### 3. Implementation Steps
1.  **Project Initialization:** Create the directory structure and basic HTML boilerplate.
2.  **Styles Implementation:** Set up CSS variables for colors and fonts; implement the dark theme.
3.  **Content Population:** Write the hero text and draft three placeholder projects (to be refined with the user).
4.  **Responsiveness:** Ensure the site looks great on mobile and desktop.
5.  **GitHub Pages Setup:** (Instructional) Guidance on pushing to GitHub and enabling Pages.

## Verification & Testing
- **Visual Check:** Verify the dark theme and font rendering.
- **Responsiveness:** Test on different screen sizes using browser dev tools.
- **Links:** Ensure all project and contact links work as intended.
